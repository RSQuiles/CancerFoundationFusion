# Downstream Task Reference

All tasks follow the same plug-in pattern: a `DownstreamTask` subclass defines *what* (data, head, loss, metrics) and `BaseDownstreamRunner` handles *how* (training loop, DDP, checkpointing). Data is embedded by a **frozen** CancerFoundation checkpoint; only the lightweight prediction head is trained.

---

## 1. Cancer Type Classification (`canc_type_class`)

| | |
|---|---|
| **Config** | `cancer_annot_config_normalized.yaml` |
| **Data** | TCGA bulk RNA-seq (`.h5ad`), cohorts: BRCA, BLCA, GBM, LGG, LUAD, UCEC |
| **Input to head** | Sample embedding from `CancerFoundation.embed()` |
| **Output** | Softmax logits over cancer-type classes |
| **Head** | `EmbeddingPredHead` — Linear(emb_dim→hidden) → ReLU → Linear(hidden→n_classes) |
| **Loss** | `CrossEntropyLoss` |
| **Metrics** | Accuracy, weighted F1, macro precision, macro recall |
| **Split** | Stratified random 80/20 by cancer type |

**Notes**
- The suffix of the project ids (cohorts) are used as cancer types.

- GBM and LGG can be merged into a single class (`merge_gbm_lgg: true`); this compensates for the biological overlap between low-grade and high-grade glioma in TCGA.
- The data is log-normalised by default. Set `normalized: false` only if passing raw counts.
- Data is not generated by bulkFM; it comes directly from TCGA via Enrico's dataset.

## 2. Cell-Type Deconvolution (`deconv`)

| | |
|---|---|
| **Config** | `deconv_config_normalized.yaml` |
| **Data** | Pseudobulk h5ad (simulated from single-cell data); cell-type proportions in `adata.obs` |
| **Input to head** | Sample embedding |
| **Output** | Softmax-normalised cell-type proportion vector |
| **Head** | `EmbeddingPredHead` — same 2-layer MLP, output_dim = n_cell_types |
| **Loss** | `KLDivLoss(reduction="batchmean")` — treats predictions as a probability distribution |
| **Metrics** | MAE, MSE, RMSE on predicted vs true proportions |
| **Split** | By context variable (e.g. study) or random 80/20 |

**Notes**
- KL divergence is applied to softmax(logits) and the normalised target proportions. The head does not include a softmax; `compute_metrics` applies `F.softmax` before computing MAE/MSE.
- If `split_by_context: true` and a `context` column is present in `adata.obs`, the test set is formed from the last 20% of unique contexts (sorted), not random samples. This tests generalisation to unseen experimental batches.
- Cell-type columns are resolved from `adata.uns["cell_type_proportion_columns"]` if present — a dict `{cell_type_name: obs_column}`. Falls back to `cell_type_mapping` in the config, then auto-detection from `adata.obs` column names.

## 3. Survival Prediction (`survival`)

| | |
|---|---|
| **Config** | `survival_config.yaml` |
| **Data** | SurvBoard (Wissel, Boeva et al. 2025): TCGA, ICGC, METABRIC, TARGET cohorts |
| **Source** | `{survboard_data_dir}/{COHORT}_{CANCER}.parquet` (e.g. `TCGA_BRCA.parquet`) |
| **Input to head** | Sample embedding |
| **Output** | Scalar log-risk score (higher = shorter predicted survival) |
| **Head** | `EmbeddingPredHead(output_dim=1)` |
| **Loss** | `CoxPHLoss` — negative partial log-likelihood (Cox Proportional Hazards) |
| **Metrics** | Harrell's C-index (uses `lifelines` if available, numpy fallback otherwise) |
| **Split** | Pre-made SurvBoard CV folds (see below) or stratified random 80/20 by cancer type |

**Pre-made splits**

When `splits_dir` is set, the loader looks for:
```
{splits_dir}/{COHORT}/{CANCER}_train_splits.csv
{splits_dir}/{COHORT}/{CANCER}_test_splits.csv
```
Each CSV has no header or row index. Each **row** is one CV fold; each value in that row is an integer row index into the corresponding parquet file. `fold_index` (default 0) selects which row to use. This is done independently per (cohort, cancer) block; a running offset maps local indices to the global concatenated position. If any file is missing for any block, the whole run falls back to stratified random split.

**Cox loss details**

The Cox partial log-likelihood for a batch of risk scores `h_i` and survival labels `(t_i, e_i)` is:

```
PLL = Σ_{i: e_i=1} [ h_i - log( Σ_{j: t_j ≥ t_i} exp(h_j) ) ]
```

The inner log-sum-exp is computed over a descending-time-sorted prefix using `torch.logcumsumexp` — this is O(n) and numerically stable. The loss returns 0 for batches with no observed events (e.g., all samples censored).

**Concordance index (C-index)**

A value of 0.5 corresponds to random ordering; 1.0 is perfect. For comparison with the SurvBoard leaderboard, use `splits_dir` with the pre-made folds and report C-index per cancer type.

**Multi-cohort / multi-cancer runs**

Set `cohorts` and `cancer_types` to lists; all found `{COHORT}_{CANCER}.parquet` files are concatenated. The same cancer type can appear in multiple cohorts (e.g. TCGA_BRCA + METABRIC_BRCA) and each block gets its own split loaded independently.

**Recommended batch size**

Each batch must contain enough observed events for the Cox loss to produce a meaningful gradient. A batch with zero events returns a zero loss. Use `batch_size ≥ 64` and ensure the training set has a reasonable event rate (≥ 20–30%).

## 4. Proteome Prediction (`proteome_pred`)

| | |
|---|---|
| **Config** | `proteome_pred_config.yaml` |
| **Data** | CPTAC (10 cancer types): paired bulk RNA-seq + proteomics (log2-ratio TMT) |
| **Source** | `bulkFM-data/data/processed/cptac/` |
| **Input to head** | Sample embedding |
| **Output** | Vector of log2-ratio protein abundances (one value per protein) |
| **Head** | `EmbeddingPredHead(output_dim=n_proteins)` |
| **Loss** | `MSELoss` — direct regression on log2-ratio abundances |
| **Metrics** | Mean Pearson R per protein, median Pearson R per protein, overall RMSE |
| **Split** | Stratified random 80/20 by cancer type (no pre-made splits for CPTAC) |

**Data loading details**

- Only samples present in **both** `rna_expression.parquet` and `protein_expression.parquet` are used (inner join on sample ID).
- `gene_protein_mapping.csv` has three columns: (0) gene symbol, (1) `has_rna` (bool), (2) `has_protein` (bool).
  - Genes with `has_rna=True` are selected as RNA input features (must also be present as columns in `rna_expression.parquet`).
  - Genes with `has_protein=True` are predicted targets (must also be present as columns in `protein_expression.parquet`).
  - The two sets are independent — a gene can have RNA data only, protein data only, or both.
- Gene symbols in CPTAC use current HGNC convention (e.g. `A1BG`, `A1BG-AS1`). The CancerFoundation vocabulary mixes HGNC with older Ensembl lncRNA names (`RP11-...`, `AC...`). `_translate_gene_symbols()` is a placeholder for a future alias-mapping step; currently it is a pass-through, and unmatched genes are silently dropped by `embed()` during its internal gene intersection.

**Metric interpretation**

A mean Pearson R around 0.3–0.5 is typical for transcript-only predictors. Values above 0.5 suggest the model captures post-transcriptional regulatory signal embedded in the pan-cancer pretraining.

Proteins with near-zero variance in either predictions or targets are excluded from the Pearson R average (the correlation is undefined/meaningless in those cases). The count of evaluated proteins is reported as `n_proteins_evaluated`.

## 5. Drug Sensitivity Prediction (`drug_sensitivity`)

| | |
|---|---|
| **Config** | `drug_sensitivity_config.yaml` |
| **Data** | BeatAML2 (Bottomly et al., Cancer Cell 2022): AML specimens with RNA-seq + drug AUC |
| **Source** | `bulkFM-data/data/processed/beataml/` |
| **Input to head** | Sample embedding |
| **Output** | AUC vector (one value per drug), range [0, 1] |
| **Head** | `EmbeddingPredHead(output_dim=n_drugs)` |
| **Loss** | `MaskedMSELoss` — MSE on sigmoid(logits) vs AUC, NaN entries excluded |
| **Metrics** | Mean Pearson R per drug, median Pearson R per drug, RMSE on non-NaN entries |
| **Split** | By **patient** (not specimen), to prevent leakage across multi-specimen patients |

**Sparse targets**

Not every drug is tested on every specimen. NaN encodes "not tested". NaN values must not contribute gradients during training and must not enter metric calculations. The implementation handles this in two places:

1. `MaskedMSELoss` computes `mask = ~torch.isnan(targets)` and applies MSE only on `pred_auc[mask]` vs `targets[mask]`.
2. `compute_metrics` similarly iterates over drugs and skips NaN entries per drug before computing Pearson R.

Because NaN is preserved in the dataset, the standard `batch[1].to(device)` call in `BaseDownstreamRunner` works correctly — float NaN is a valid float32 value.

**Drug filtering**

Drugs tested on fewer than `min_drug_samples` specimens (default 20) are dropped before training. The rationale: with very few test points, a drug's gradient contribution per step is noisy, and its per-drug correlation at evaluation is unreliable. Reducing `min_drug_samples` includes more drugs at the cost of noisier training.

**Patient-level split**

BeatAML2 contains patients with multiple specimens (different time points or conditions). Splitting at specimen level would allow the model to implicitly see a patient's gene expression profile at training time, then predict that same patient's drug response at test time — an overly optimistic setup. The implementation loads `patient_id` from `metadata.csv` and guarantees all specimens from the same patient are in the same partition.

**Sigmoid convention**

The head outputs raw logits in (−∞, +∞). `MaskedMSELoss` applies sigmoid before MSE so that the gradient space is in [0,1] matching the AUC targets. `compute_metrics` applies the same sigmoid transformation before computing Pearson R and RMSE. This must be kept consistent between the two.

## Shared Utilities

### `parquet_to_adata(df, gene_cols)`

Converts a bulk RNA-seq Parquet DataFrame (samples × genes) to an `AnnData` object so that `CancerFoundation.embed()` can process it. Used by all three bulk-data tasks (Survival, Proteome, Drug Sensitivity). Non-gene columns (e.g. `OS_days`, `cancer_type`) must be excluded from `gene_cols` before calling.

### `CoxPHLoss`

Implements the negative Cox partial log-likelihood as a PyTorch loss module. Input: `risk` tensor (N,) or (N,1) of scalar risk scores and `targets` tensor (N, 2) with `[OS_days, OS_event]`. Numerically stable via `torch.logcumsumexp` after sorting by descending event time.

### `MaskedMSELoss`

MSE loss that ignores NaN positions in `targets`. Applies `torch.sigmoid` to predictions before computing MSE so predictions are in the same [0, 1] range as AUC targets.

## Adding a New Task

1. Copy `task_template.py` and implement the 6 abstract methods.
2. Decorate with `@TaskRegistry.register`.
3. Import the new class in `run_downstream_task.py` (or any script that calls `TaskRegistry.get_task`).
4. Create a YAML config under `evaluate/finetune/`.
5. Run:
   ```bash
   python evaluate/finetune/run_downstream_task.py --config evaluate/finetune/my_task_config.yaml
   ```
